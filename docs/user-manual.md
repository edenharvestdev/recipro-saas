# คู่มือใช้งาน Recipro (recipro.love)

> ระบบคิดต้นทุน + สต๊อก + POS + บัญชี + เมนู QR สำหรับร้านอาหาร/เครื่องดื่ม
> เข้าที่ **www.recipro.love** · ล็อกอินด้วยอีเมล/รหัสที่ตั้งไว้

---

## 1. หมวดสินค้า — ตัวคุมการตัดสต๊อก (สำคัญสุด)
เมนู **สต๊อก/วัตถุดิบ** → เพิ่ม/แก้วัตถุดิบ → ช่อง **"หมวดสินค้า"** (ช่องเดียว) คุมทั้งการขายและการตัดสต๊อก:

| เลือกหมวด | ผลที่เกิด |
|---|---|
| **ของขาย (SALE)** | ขายใน POS ได้ทันที (ขึ้นช่องราคา) |
| วัตถุดิบ / ของเตรียม / ของผสม / กึ่งสำเร็จ | เป็น "ของใช้ทำ" — ตัดสต๊อกเมื่อถูกใช้ในสูตร |
| บรรจุภัณฑ์ | ตัดตอนขาย |
| ของใช้สิ้นเปลือง | ตัดรายวัน/เบิก |
| ของใช้ถาวร (Asset) | **ไม่ตัดสต๊อก** (เครื่องมือ/ทรัพย์สิน) |

- ปล่อย "ยังไม่ระบุ" ไว้ = ของเดิมไม่เปลี่ยนสถานะ (ค่อย ๆ จัดได้)
- ช่อง "หมวดหมู่" (พิมพ์เอง) = แค่ป้ายจัดกลุ่ม ไม่เกี่ยวสต๊อก
- ใส่ **รูปสินค้า** ได้ (ใช้โชว์ใน POS + เมนู QR)

## 2. สูตร / เมนู
เมนู **สูตร/เมนู** → สร้างสูตร: ใส่ชื่อ ราคา ส่วนผสม (เลือกวัตถุดิบ + จำนวน) → **ต้นทุน/กำไรคำนวณสด**
- **โคลนสูตร** (ปุ่มไอคอนคน): ก๊อปเมนูเดิม → แก้เป็นเมนูใหม่ได้เลย (สต๊อกเริ่มที่ 0)
- ส่วนผสมใช้ "ของเตรียม/ของกลาง" (สูตรซ้อน) ได้

## 3. ขายหน้าร้าน (POS) → ตัดสต๊อกอัตโนมัติ
เมนู **ขาย & ออนบิล** → กดเมนู → ตะกร้า → ชำระเงิน
- ระบบ **ตัดวัตถุดิบตามสูตรอัตโนมัติ** (ผ่าน engine ที่รู้หมวด) + ออกบิล
- Asset ไม่ถูกตัด · ตัวเลือก (options) ที่ใช้วัตถุดิบเพิ่มก็ตัดด้วย

## 4. ป้ายราคา · เตือนสั่งของ
- **ป้ายราคา**: หน้าวัตถุดิบ → ปุ่ม "ป้ายราคา" → พิมพ์ป้ายติดสินค้า
- **เตือนสั่งของ**: ตั้ง "จุดสั่งซื้อ (low stock)" ในวัตถุดิบ → ของที่ถึงจุดจะเตือน (มีลิงก์สั่งซื้อ)

## 5. บัญชี (หน้า "รายจ่าย" / รายรับ-รายจ่าย)
- **รายรับ-รายจ่าย**: เพิ่มได้ตรง ๆ + สรุปกำไรสุทธิ + filter ช่วงวัน
- **รายจ่ายประจำ** 🔁: ตั้งค่าเช่า/เงินเดือน/ค่าน้ำไฟครั้งเดียว → ปุ่ม "ลงรายจ่ายประจำเดือนนี้" (แก้ยอดได้) + เตือนรายเดือน
- **เงินสดย่อย** 💵: กด "เปิดใช้" → เติมเงินก้อน → โชว์ยอดคงเหลือ + เตือนใกล้หมด (รายจ่ายที่จ่าย "เงินสดย่อย" จะตัดจากก้อนนี้)
- **แนบสลิป**: ทุกรายจ่ายแนบรูปสลิปได้ (ไว้รวมยื่นภาษี)

## 6. กระทบยอด Delivery (⚠️ อ่านให้เข้าใจ)
หน้า **รายจ่าย** → การ์ด **"กระทบยอด Delivery"** → กรอก: Platform · วันที่ · **ยอดขายรวม** · **เงินเข้าจริง** → กดบันทึก

ระบบลงให้อัตโนมัติ:
- **รายรับ = ยอดขายรวม** · **ค่าธรรมเนียม platform = ยอดขาย − เงินเข้าจริง** (กำไรสุทธิ = เงินเข้าจริง)

**❗ จุดที่มักงง — เรื่องสต๊อก:**
- การกรอกนี้จัดการ **"เงิน" อย่างเดียว — ไม่ตัดสต๊อก** และไม่ผูกกับบิล
- เหตุผล: platform หัก GP+โฆษณา+คูปองรวมกัน คิดรายแก้วไม่ได้ → จึงกรอกเงินทีเดียว "ต่อวัน"
- **การตัดสต๊อก delivery ตอนนี้**: ยังไม่อัตโนมัติ — ถ้าต้องการให้ตัด มี 2 ทาง:
  - (ก) ขายออเดอร์ delivery ผ่านหน้า POS ปกติ (จะตัดวัตถุดิบ) — แต่จะนับยอดขายซ้ำกับที่กรอก ต้องเลี่ยงกรอกรายรับซ้ำ
  - (ข) **[แนะนำ]** เพิ่มช่อง "เมนู + จำนวนที่ขายวันนั้น" ในการ์ด Delivery → กดบันทึกแล้วตัดสต๊อก+ต้นทุนให้พร้อมกับลงเงิน (ตรงกับที่ออกแบบไว้ "ต่อวัน+จำนวน") — *ฟีเจอร์นี้ยังไม่ได้ทำ แจ้งได้ถ้าต้องการ*

## 7. เมนู QR — ลูกค้าสั่งเอง
หน้า **รายจ่าย** → การ์ด **"เมนูออนไลน์ (QR)"** → กด **เปิดใช้**
- ได้ **QR + ลิงก์** → พิมพ์ติดร้าน/ส่งให้ลูกค้า → ลูกค้าสแกน เห็นเมนู (เฉพาะ "ของขาย") กดใส่ตะกร้า สั่ง → ได้ **หมายเลขคิว**
- ออเดอร์ลูกค้าโผล่เป็น **"คิว"** ในการ์ดนี้ (รีเฟรชหน้าเพื่ออัปเดต)
- ตอนนี้เป็นแบบ **รับที่ร้าน** (ยังไม่จ่ายออนไลน์/ยังไม่ตัดสต๊อกอัตโนมัติต่อออเดอร์ = เฟส M3.2)

- เมลเตือน delivery (ต้องตั้ง RESEND_API_KEY)
