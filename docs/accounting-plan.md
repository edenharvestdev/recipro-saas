# Accounting + Delivery Reconciliation — Review & Plan

> สถานะ: **review/plan เท่านั้น — ยังไม่แก้โค้ด** รอเจ้าของปรับ
> หลัก: เพิ่มแบบ additive · เก็บข้อมูลเดิม · เปิด/ปิดโมดูลได้ (ร้านเล็กไม่ต้องใช้ครบ)

## 1. สถานะปัจจุบัน
- `expenses(id, shop_id, expense_date, category, description, amount, payment_type[cash|transfer|card|other], note, kind[expense|income])` = ledger รายรับ-รายจ่ายรวม
- หน้า "รายรับ-รายจ่าย": สรุป รายรับ/รายจ่าย/สุทธิ + filter ช่วงวัน/หมวด/ประเภท + เพิ่มรายรับ/รายจ่าย
- `bills(number, doc_type, items_json, status, discount, tax)` = บิลขาย/เอกสาร (มี doc type "ใบส่งของ")
- **ขาด**: รายจ่ายประจำ (recurring), float เงินสดย่อย (ยอดคงเหลือ), แนบสลิป, delivery reconciliation

---

## 2. ส่วนที่ 1 — ระบบบัญชีรวม (รายรับ-รายจ่าย + กระแสเงินสด)

### 2.1 รายจ่ายประจำ (recurring) — [เจ้าของเคาะ: ล็อกเทมเพลต + แก้ตัวเลขได้ + เตือน]
ตารางใหม่ `recurring_expenses(id, shop_id, name, category, default_amount, day_of_month, active)`
- ตั้งครั้งเดียว (ล็อกไว้): ค่าเช่า 8000 (ทุกวันที่ 1), เงินเดือน, ค่าน้ำ/ไฟ ฯลฯ — เป็น **เทมเพลตยอดเดิม**
- ถึงรอบ (วันที่กำหนด) → **เตือนจ่ายประจำเดือน** → หน้าทบทวน: โชว์ยอด default + **แก้ตัวเลขได้** (บางเดือนค่าน้ำไฟไม่เท่ากัน) → กดยืนยัน → insert เข้า `expenses`
- กันลงซ้ำเดือนเดียวกัน (`source_recurring_id` + เดือน) · ลงเป็น expense ปกติ + ป้าย "ประจำ"
- = ได้ทั้ง "ล็อกยอดเดิม" + "แก้รายเดือน" + "เตือน" ในตัวเดียว

### 2.2 กระแสเงินสดย่อย / petty cash float (เปิด/ปิดได้)
แนวคิด: เงินสดก้อน (เช่น 2000) เติมเข้า แล้วค่อย ๆ ตัดจ่ายรายวัน → โชว์**ยอดคงเหลือ**
- ตารางใหม่ `cash_topups(id, shop_id, date, amount, note)` = การเติมเงินก้อน
- รายจ่ายที่ `payment_type='cash'` = ตัดจากก้อนนี้
- **ยอดคงเหลือ = Σ topups − Σ cash expenses** (โชว์การ์ด "เงินสดเหลือ ฿X")
- เตือนเมื่อใกล้หมด · toggle ใน settings (`use_petty_cash`) — ปิดไว้ร้านที่ไม่ใช้
- ตอนเติมรอบใหม่ = topup ใหม่ (ก้อนเดิมหมด/ทบยอดได้)

### 2.3 แนบสลิปทุกรายจ่าย (เพื่อยื่นภาษี)
- เพิ่มคอลัมน์ `expenses.slip_data text` (base64 รูป) — เหมือน `recipes.img_data`
- ฟอร์มรายจ่าย: อัปโหลด/ถ่ายรูปสลิป
- ฟีเจอร์ **"ส่งออกรายจ่าย + สลิป"** (ช่วงวันที่ → PDF/zip) สำหรับยื่นภาษี/บัญชี

## 3. ส่วนที่ 2 — ทางลัด Delivery (จุดขายการตลาด)

**ปัญหา:** platform (Grab/LINE MAN/Foodpanda) หัก GP/ค่าธรรมเนียมซับซ้อน คิดรายแก้วไม่ได้ → ทำ **reconciliation รายวัน + จำนวน**

### Flow (state machine ของบิล delivery)
```
[ขาย] ──► บิล delivery (status = ค้างรับ/pending_payout)         ← ออกทุกออเดอร์ ระบุ platform + ยอดหน้าบิล(face) + จำนวน
   │       (ยังไม่ตัดสต๊อก · ยังไม่รับรู้รายรับ)
   │
[วันถัดไป] เมลเตือน "กรอกยอด delivery เมื่อวาน"
   │
[กรอกยอดจริง] ──► ใส่ "เงินเข้าจริง" (net หลังหัก GP) ต่อ platform/วัน
   │       ระบบทำให้อัตโนมัติ:
   │         1) ยืนยันบิลของวันนั้น → ตัดสต๊อก + คำนวณ COGS (จากสูตร/Item Master)
   │         2) ส่วนต่าง (face − เงินเข้าจริง) = ค่า GP/ธรรมเนียม → ลงเป็น "บิลส่วนลด"/ค่าใช้จ่าย platform
   │         3) รับรู้รายรับ = เงินเข้าจริง
   ▼
[ปิดยอดวัน] กำไร = เงินเข้าจริง − COGS − ค่าใช้จ่ายอื่น
```

### โมเดลข้อมูล (additive) — [อัปเดตตามที่เจ้าของเคาะ]
- `bills` เพิ่ม: `channel text` ('store'|'delivery'), `platform text` ('grab'|'lineman'|'foodpanda'|...), `payout_status text` ('pending'|'reconciled'), `face_amount numeric`, `net_payout numeric`, `recon_date date`
- ตารางใหม่ `delivery_payouts(id, shop_id, platform, biz_date, gross_face, net_received, fee_amount, bill_ids jsonb, reconciled_at)` = สรุปการกระทบยอดต่อ platform/วัน
- **ตัดสต๊อก = ตอนออกบิล** (real-time, เคาะแล้ว) ผ่าน engine `/api/pos/sell` (P3) → COGS รู้ทันทีตั้งแต่ออกบิล · การ reconcile วันถัดไป**จัดการเฉพาะฝั่งเงิน** (รายรับ net + ค่าหัก)

### จุดสำคัญ — [เจ้าของเคาะ]
- **ค่าหัก platform = เลขเดียว (lump)** ✅ ผู้ใช้กรอกแค่ **"เงินเข้าจริง"** ต่อ platform/วัน → ระบบคิดส่วนต่าง = `Σface − net` เป็นก้อนเดียว ลงเป็น **"ค่าใช้จ่าย platform"** + โชว์ % effective (ครอบคลุม GP+โฆษณา+คูปอง+ค่าส่ง — ไม่ต้องแยก)
  - (ออปชันอนาคต: ช่องแยกย่อย GP/ads/คูปอง สำหรับร้านที่อยากละเอียด — default ปิด)
- **ตัดสต๊อกตอนออกบิล** ✅ → สต๊อก real-time
- **ออเดอร์ยกเลิก/คืนเงิน**: ต้องมีปุ่ม "ยกเลิกบิล" → คืนสต๊อก (add-back) ก่อน reconcile (edge case ที่ต้องทำ เพราะตัดสต๊อกไปแล้วตอนออกบิล)
- **คิดต่อวัน+จำนวน**: reconcile = ผลรวม qty + face ของวัน, ค่าหักเป็นก้อนเดียว/วัน (ไม่กระจายรายแก้ว)

## 4. การแบ่งเฟส
- **A1** รายจ่ายประจำ (ตาราง + ปุ่มลงประจำเดือน)
- **A2** petty cash float (เติม/คงเหลือ/เตือน + toggle)
- **A3** แนบสลิป + ส่งออกภาษี
- **A4** delivery: บิลค้างรับ + ระบุ platform/face
- **A5** delivery: หน้า reconcile รายวัน → ตัดสต๊อก+COGS+บิลส่วนลด
- **A6** เมลเตือนกรอกยอด delivery (ผูก RESEND/SMTP)
